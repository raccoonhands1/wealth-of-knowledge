>[!definition]
> the data path is the path that data takes in a controled sequence

>[!info]
> each data path has:
> 1. IF: Instruction fetch
> 2. ID: Instruction decode and register file read
> 3. EX: Execution or address calculation
> 4. MEM: Data memory access
> 5. WB: Write back
